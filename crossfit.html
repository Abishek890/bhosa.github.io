    FUZZ METHODOLOGY

  • The basic idea that we are gonna use in [[CROSSFIT]] is the following:

    1. SO WHEN WE SENT A GET REQUEST TO SOME WEBSITE,WE ALSO SENT A ORIGIN HEADER .This is to help the server understand from where the request is coming.
    2. THE SERVER WEBSITE CHECKS THE ORIGIN AND IF THE WEBSITE IS ALLOWED THEN IT SENTS A Access-Control-Allow-Origin: http://ftp.crossfit.htb HEADER TO SPECIFY THAT THE DOMAIN FROM WHICH THE REQUESTS ARE ORIGINATING CAN HAVE ACCESS TO THE RESOURCES IN THE SERVER.
    3. ==WHAT IS OUR OBJECTIVE?==\
    4. IT IS TO FIND REAL SUB-DOMAINS SO THAT WE CAN FURTHUR ENUMERATE.
    5. AND WE DO THAT WITH THE HELP OF THE RESPONSE HEADER-Access-Control-Allow-Origin:
    6. WE USE FFUF TO SEND A GET REQUEST WITH RANDOM SUB-DOMAINS FROMA WORDLIST AND THEN FILTER THE RESPONSES ON THE BASIS OF THE HEADER Access-Control-Allow-Origin: USINF A REGEX OPTION IN FFUF(same can be done in wfuzz).
    7. The servers wont send a Access-Control-Allow-Origin: header if the origin is some domain or sub-domain that it does not recognize.
